As the sole developer on the frontend interface of **Daily Dose**, a tiffin meal subscription platform, one of our key product requirements was a clean "Order Status Tracker" screen. We wanted our 150+ daily active users to see exactly when their meals moved from preparation to dispatcher delivery.

Here is a look behind the scenes at how we modeled the tracking status database tables and integrated state triggers.

## 1. The Database Schema Model in MongoDB

In MongoDB, we wanted to ensure order records were audit-ready and kept track of timestamps for each status transition. We designed an `Order` model with a structured array of status logs:

```javascript
const OrderSchema = new mongoose.Schema({
  user: { type: mongoose.Schema.Types.ObjectId, ref: 'User', required: true },
  items: [{
    name: String,
    quantity: Number,
    price: Number
  }],
  totalAmount: Number,
  status: {
    type: String,
    enum: ['placed', 'preparing', 'out-for-delivery', 'delivered', 'cancelled'],
    default: 'placed'
  },
  statusLog: [{
    status: String,
    timestamp: { type: Date, default: Date.now }
  }]
}, { timestamps: true });
```

Keeping a historical log of status changes in the `statusLog` sub-document array allowed us to render an interactive step-by-step progress timeline on the client.

## 2. API Endpoint Design

We exposed a clean GET endpoint for fetching the order history. The controller parses the status log and returns it sorted so the client gets the order journey immediately.

```javascript
// Express.js Route Controller
router.get('/orders/:id/tracking', async (req, res) => {
  try {
    const order = await Order.findById(req.params.id)
      .select('status statusLog')
      .lean();

    if (!order) {
      return res.status(404).json({ error: 'Order not found' });
    }

    res.json({
      currentStatus: order.status,
      history: order.statusLog
    });
  } catch (err) {
    res.status(500).json({ error: 'Internal server error' });
  }
});
```

## 3. Polling vs. WebSockets: The Architectural Decision

While WebSockets (`socket.io`) are often touted as the default choice for "real-time" apps, they come with substantial maintenance costs: handling reconnect sockets, managing state across server nodes, and maintaining active TCP connections.

Given that Daily Dose orders change state at most 3 to 4 times over a 1-hour window, WebSockets would be overkill. 

Instead, I implemented a **smart polling strategy** on the client:
- Poll the endpoint every 15 seconds.
- Halt polling if the status is resolved (e.g. `delivered` or `cancelled`).
- Only start polling when the tracking page is actively focused.

This saved CPU cycles and database connections while providing a real-time feel.

## 4. React Polling State Handler

On the React client, I wrote a custom effect handler to manage active intervals safely, clearing timers when components unmount or statuses resolve.

```tsx
import { useEffect, useState } from 'react';

export function useOrderTracking(orderId: string) {
  const [trackingData, setTrackingData] = useState<any>(null);

  useEffect(() => {
    let intervalId: NodeJS.Timeout;

    async function fetchStatus() {
      try {
        const res = await fetch(`/api/orders/${orderId}/tracking`);
        const data = await res.json();
        setTrackingData(data);

        // Stop polling if order has reached final state
        if (data.currentStatus === 'delivered' || data.currentStatus === 'cancelled') {
          clearInterval(intervalId);
        }
      } catch (err) {
        console.error('Failed to load tracking data', err);
      }
    }

    // Run first fetch immediately
    fetchStatus();

    // Start interval
    intervalId = setInterval(fetchStatus, 15000);

    return () => clearInterval(intervalId);
  }, [orderId]);

  return trackingData;
}
```

## Conclusion

This approach allowed us to launch a reliable tracker within days, providing an instant visual roadmap of tiffin dispatch dates. In a future iteration, we could introduce server-sent events (SSE) for a push-based interface with minimal architectural disruption.
